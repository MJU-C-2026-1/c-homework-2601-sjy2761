# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 스마트 벌크업 및 영양 목표 관리 시뮬레이터 (Smart Bulk-up & Nutrition Planner)
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 효과적인 근성장과 체중 증량을 목표로 하는 사용자를 위한 **'종합 벌크업 관리 시스템'**입니다.
*  사용자는 자신의 이름 이니셜과 현재 체중, 목표 체중을 입력하여 증량에 필요한 수치를 확인하는 것으로 시작합니다.
*   시스템은 입력된 신체 데이터를 바탕으로 하루 권장 칼로리와 3대 영양소(단백질, 탄수화물, 지방)의 목표 섭취량을 자동으로 산출합니다.
*   이후 개발 단계에 따라, 매일 섭취한 영양소를 입력받아 목표 대비 달성률을 판별하고(조건문), 메뉴 시스템을 통해 식단과 운동을 반복적으로 기록할 수 있도록 구성할 예정입니다(반복문).
*   최종적으로는 일주일간의 체중 변화 기록을 배열에 저장하고 함수를 통해 주간 변화 추이를 분석하는 기능을 포함하여, 사용자의 체계적인 신체 변화를 돕는 완성도 높은 소프트웨어를 개발할 계획입니다.

## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 사용자 신체 스펙 및 목표 입력: 이니셜, 현재 체중, 목표 체중 입력 기능
 
2. 벌크업 목표 자동 산출: 증량 목표 무게 및 일일 권장 칼로리/영양소 계산

3. 영양 섭취 상태 판별: 목표 대비 현재 섭취량의 과잉/부족 여부 판단 (Phase 2)

4. 데일리 로그 관리 메뉴: 식단 입력, 운동 기록, 현재 상태 조회 등을 선택하는 메뉴 시스템 (Phase 3)

5. 주간 변화 리포트: 7일간의 체중 및 영양 기록 데이터 저장 및 요약 출력 (Phase 4)

6. 프로그램 종료: 데이터 저장 및 시스템 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * 사용자로부터 이름(이니셜), 현재 체중(실수), 목표 체중(실수)을 입력받아 벌크업에 필요한 영양 수치를 계산하여 출력합니다.
  *문자형(char), 실수형(float), 정수형(int) 변수를 모두 활용하여 데이터 특성에 맞는 최적의 자료형을 설계했습니다.
  *사칙연산을 활용해 증량 목표 무게와 하루 권장 칼로리, 3대 영양소(단백질/탄수화물/지방) 목표치를 자동으로 산출하는 로직을 구현했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 → 조건문 → 반복문 → 배열/함수)에 적합한 헬스 및 벌크업 관리 시뮬레이터 시나리오 기획 요청"
    * **적용 내용:** 단순 무게 계산기를 넘어, '기초 정보 입력(1차)' → '영양 상태 판별(2차)' → '식단/운동 메뉴 시스템(3차)' → '주간 변화 리포트 및 함수화(4차)'로 이어지는 체계적인 확장 로드맵을 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개 이상 사용)을 충족하기 위한 데이터 구조 고도화 논의"
    * **적용 내용:** 단순히 체중만 다루는 대신, 벌크업의 핵심인 3대 영양소(단백질, 탄수화물, 지방) 변수를 추가 선언하여 총 8개의 변수를 활용함. 이를 통해 기획의 완성도를 높이고 과제 조건을 완벽히 충족함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 체중 소수점이 너무 길게 출력되어 가독성이 떨어짐.
     * **해결:** printf("%.1f", weight);와 같이 서식 지정자를 수정하여 소수점 첫째 자리까지만 깔끔하게 출력되도록 개선함.
  2. **문제 2:** scanf로 몸무게 입력 시 엉뚱한 값(쓰레기 값)이 저장되는 현상 발생.
     * **해결:** 실수형(float) 변수인 currentWeight에 값을 입력받으면서, 무의식적으로 정수형 서식 지정자인 %d를 사용했던 것이 원인이었음, 자료형과 서식 지정자의 짝을 정확히 맞춰야 한다는 점을 복습하고, %d 대신 %f로 수정하여 데이터를 정상적으로 입력받음.
