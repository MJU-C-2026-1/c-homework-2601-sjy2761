# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 스마트 벌크업 및 영양 목표 관리 시뮬레이터 (Smart Bulk-up & Nutrition Planner)
<br>


## 2. 시나리오 (5~10줄)
* 본 프로그램은 효율적인 체중 증량을 목표로 하는 사용자를 위한 '단계별 벌크업 관리 시스템'의 기초 단계입니다.
* 사용자의 이니셜과 현재/목표 체중, 달성 기간을 입력받아 벌크업에 필요한 핵심 영양 지표와 운동 목표를 산출합니다.
* 1차 과제에서는 기초 입출력과 산술 연산을 구현하며, 향후 과제를 통해 영양 상태 판별(조건문), 메뉴 시스템(반복문), 주간 데이터 관리(배열/함수) 기능을 순차적으로 확장하여 최종적인 건강 관리 소프트웨어를 완성할 계획입니다.
  

## 3. 예상 기능 및 메뉴 (최소 5개)
1. 사용자 프로필(이니셜) 설정 기능

2. 신체 데이터를 기반으로 한 증량 목표 무게 산출

3. 목표 달성 D-Day 설정 및 관리

4. 체중별 하루 권장 칼로리 및 단백질 섭취량 가이드 제공

5. 일일 권장 운동 시간 설정 및 출력

6. 프로그램 종료: 데이터 저장 및 시스템 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
* 사용자로부터 이니셜(문자), 현재/목표 체중(실수), 목표 기간(정수)을 입력받는 화면을 구현했습니다.

* 사칙연산을 활용해 증량 목표와 하루 권장 영양 수치를 계산하여 출력했습니다.

* char, int, float의 3가지 자료형과 8개의 변수를 사용하여 데이터 구조의 완성도를 높였습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 → 조건문 → 반복문 → 배열/함수)에 적합한 헬스 및 벌크업 관리 시뮬레이터 시나리오 기획 요청"
    * **적용 내용:** 단순 무게 계산기를 넘어, '기초 정보 입력(1차)' → '영양 상태 판별(2차)' → '식단/운동 메뉴 시스템(3차)' → '주간 변화 리포트 및 함수화(4차)'로 이어지는 체계적인 확장 로드맵을 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개 이상 사용)을 충족하기 위한 데이터 구조 고도화 논의"
    * **적용 내용:** 단순히 체중만 다루는 대신, 목표 단백질량, 목표 운동시간, 목표 달성 기간 등 변수를 추가 선언하여 총 8개의 변수를 활용함. 이를 통해 기획의 완성도를 높이고 과제 조건을 완벽히 충족함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 체중 소수점이 너무 길게 출력되어 가독성이 떨어짐.
     * **해결:** printf("%.1f", weight);와 같이 서식 지정자를 수정하여 소수점 첫째 자리까지만 깔끔하게 출력되도록 개선함.
  2. **문제 2:** scanf로 몸무게 입력 시 엉뚱한 값(쓰레기 값)이 저장되는 현상 발생.
     * **해결:** 실수형(float) 변수인 currentWeight에 값을 입력받으면서, 무의식적으로 정수형 서식 지정자인 %d를 사용했던 것이 원인이었음, 자료형과 서식 지정자의 짝을 정확히 맞춰야 한다는 점을 복습하고, %d 대신 %f로 수정하여 데이터를 정상적으로 입력받음.
